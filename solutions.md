# Тестовое задание: Публикация товара на маркетплейсе (витрине товаров).
## User Story
Как продавец на маркетплейсе, я хочу управлять карточками товаров, чтобы поддерживать актуальную информацию о продуктах и предлагать их покупателям
### Категории приемки
- [ ] Заполнены все обязательные поля для карточки товара (изображение, название, цена, краткое описание, категория, количество на складе) 
- [ ] Цена товара указана и она больше 0
- [ ] Указано количество на складе
- [ ] Кнопка "Отправить" доступна и отправляет данные на модерацию
- [ ] После успешной публикации товар отображается на витрине

## Use Case 1: Создание карточки товара
|Элемент|Значение|
|:---|:---|
|**ID**|UC-01|
|**Название**|Создание карточки товара|
|**Акторы**|Основной: Продавец, Система: Интернет магазин|
|**Предусловия**|Продавец авторизован в системе. У него есть права на добавление товара. Карточка товара заполнена корректно|
|**Триггер**|Продавец нажимает кнопку "Опубликовать" на странице черновика товара|

### Основной поток
1. Продавец переходит в раздел управления товарами
2. Продавец нажимает "Добавить товар"
3. Система отображает форму создания карточки товара
4. Продавец вводит данные о товаре (изображение, название, цена, краткое описание, категория и т.д)
5. Продавец подтверждает действие нажимая "Сохранить"
6. Система проверяет корректность введенных данных о товаре (цена и остаток товара больше 0, все поля карточки товара заполнены)
7. Если данные корректны, система сохраняет карточку и добавляет ее в список товаров продавца со статусом "Опубликован", и товар становится виден на витрине

### Альтернативный поток: продавец вводит некорректные данные 
1. Система уведомляет об ошибке (например, "Цена должна быть больше нуля")
2. Продавец исправляет данные и повторяет попытку сохранения

### Исключения: Сбой при сохранении данных
1. Система отображает сообщение об ошибке и предлагает повторить попытку

## Use Case 2: Просмотр карточек товаров
|Элемент|Значение|
|:---|:---|
|**ID**|UC-02|
|**Название**|Просмотр карточек товаров|
|**Акторы**|Основной: Продавец, Система: Интернет магазин|
|**Предусловия**|Продавец авторизован в системе. У него есть хотя бы одна созданная карточка товара|
|**Триггер**|Продавец нажимает кнопку "Мои товары"|

### Основной поток
1. Продавец заходит в раздел управления товарами
2. Система отображает список карточек товаров с основными данными (изображение, название, цена, краткое описание, категория, количество на складе)
3. Продавец выбирает конкретную карточку для просмотра
4. Система отображает подробную информацию по выбранной карточке товара

### Альтернативный поток: Список товаров пуст
1. Система уведомляет продавца о том, что товаров нет
2. Система предлагает создать первую карточку товара

### Исключения: Система отображает сообщение об ошибке загрузки содержимого страницы
1. Продавцу предлагается повторить действие позже

## Use Case 3: Изменение данных карточки товара
|Элемент|Значение|
|:---|:---|
|**ID**|UC-03|
|**Название**|Изменение данных карточки товара|
|**Акторы**|Основной: Продавец, Система: Интернет магазин|
|**Предусловия**|Продавец авторизован в системе. Карточка товара существует|
|**Триггер**|Продавец нажимает кнопку "Настройка" рядом с карточкой товара|

### Основной поток
1. Продавец выбирает нужную карточку
2. Продавец заходит в настройки карточки товара
3. Система отображает форму настроек с данными на текущий момент
4. Продавец вносит необходимые изменения
5. Продавец подтверждает свои действия с помощью кнопки "Сохранить"
6. Если данные верны, система обновляет карточку товара

### Альтернативный поток: Продавец ввел некорректные данные
1. Система уведомляет об ошибке и предлагает исправить данные
2. Продавец исправляет данные и повторяет попытку внести именения в карточку товара

### Исключения: Сбой при обновлении данных
1. Система отображает сообщение об ошибке и предлагает повторить попытку позже

## Use Case 4:  Удаление карточки товара
|Элемент|Значение|
|:---|:---|
|**ID**|UC-04|
|**Название**|Удаление карточки товаров|
|**Акторы**|Основной: Продавец, Система: Интернет магазин|
|**Предусловия**|Продавец авторизован в системе. Карточка товара существует|
|**Триггер**|Продавец нажимает кнопку "Настройка" рядом с карточкой товара|

### Основной поток
1. Продавец выбирает нужную карточку
2. Продавец заходит в настройки карточки товара
3. Система запрашивает подтвеждение действий
4. Продавей подтверждает удаление
5. Система удалает карточку из списка товаров

### Альтернативный поток: Продавец отменяет действие
1. Система оставляет карточку товара без изменений
   
### Исключения: Попытка удалить товар с активным заказом
1. Система не дает провести удаление и уведомляет продавца о том, что товар заказан
